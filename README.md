# Healthcare Appeals & Grievance Operations Analytics

Professional-grade healthcare operations analytics project using Databricks, SQL, and Power BI

## Business Problem
Appeals and grievance teams need a reporting layer that highlights case volume, turnaround time, backlog aging, service level agreement (SLA) risk, overturn rates, and escalation patterns. Without this visibility, leaders cannot quickly identify which issue types, plans, or regions are driving delays and compliance risk.

## Objective
Build a Databricks-based analytics workflow that:
1. ingests raw appeals operations data
2. transforms it into curated reporting tables
3. calculates KPI metrics in SQL
4. supports executive dashboards in Power BI

## Tech Stack
- Databricks
- SQL
- Power BI
- CSV
- GitHub

## Dashboard Pages
1. Executive Overview
2. Backlog & Aging
3. Quality / Resolution Outcomes
4. Team Productivity & Compliance

- Built Databricks and SQL transformation logic to analyze appeal volume, resolution patterns, escalation risk, and team productivity
- Developed Power BI dashboards to improve leadership visibility into appeals performance, compliance exposure, and operational bottlenecks
